3. GitHub Authentication

We use a Personal Access Token (fine‑grained). Recommended

Generate a token

  1. GitHub → Settings → Developer settings → Personal access tokens → Fine‑grained tokens.
  2. Click Generate new token.
  3. Set repository access to All repositories.
  4. Under Repository permissions, enable:
    • Contents → Read and write
    • Metadata → Read (always)
  5. Generate and copy the token (you won't see it again).

Cache the token (one‑time)

💡 This stores the token on your local machine – never in the repository. 
git config --global credential.helper store

Then do a one‑time push to save the token:

git push origin master:main
# Username: aminul-du
# Password: paste your token

After this, future pushes work automatically.

🔒 Alternative (more secure): Use an environment variable: 
export GITHUB_TOKEN="your_token"
./deploy.sh

6. deploy.sh – One‑Command Deployment

This script pulls, adds, commits, and pushes — no token inside.

#!/bin/bash
set -e

GREEN='\033[0;32m'
YELLOW='\033[1;33m'
NC='\033[0m'

echo -e "${GREEN}🚀 Deploying...${NC}"

git pull origin main --allow-unrelated-histories --no-edit || true
git add .
COMMIT_MSG="${1:-Auto-deploy: $(date '+%Y-%m-%d %H:%M:%S')}"
git commit -m "$COMMIT_MSG" || echo -e "${YELLOW}No changes${NC}"
git push origin master:main

echo -e "${GREEN}✅ Done.${NC}"
💡 Usage: 
chmod +x deploy.sh
./deploy.sh "Your commit message"   # or just ./deploy.sh

8. Troubleshooting

❌ Permission denied (publickey)

Switch to HTTPS:

git remote set-url origin https://github.com/aminul-du/My-RajshahiOnline-WebHost.git
❌ Push rejected (non-fast-forward)

Pull first, then push:

git pull origin main --allow-unrelated-histories
git push origin master:main
❌ Merge conflict

Keep your version:

git checkout --ours README.md
git add README.md
git commit -m "Resolved conflict"
git push
❌ GitHub push protection – token detected

The token is in your commit history. Remove it:

# Reset to last clean commit
git reset --hard 8cd91d8
# Re‑add files (without token)
git add .
git commit -m "Clean commit"
git push origin master:main --force

Then never hardcode tokens in scripts — use credential.helper store or environment variables.
